Технологии, используемые при Web-разработке

HTML - сокращение от английского Hyper Text Markup Language - язык разметки гипертекста. Так называется стандарт языка разметки документов для публикации в Интернет в виде Web страниц. Документы HTML по внутреннему содержанию являются обычными текстовыми файлами, поэтому доступны для прочтения на любой операционной системе. Язык HTML позволяет включать в документ ссылки на другие подобные документы в Сети, форматировать текст, включать таблицы, списки, фотографии, применять другие виды оформления. Современные документы HTML активно используют для оформления внешнего вида документа таблицы каскадных стилей CSS, а для создания динамических эффектов и взаимодействия с пользователем используется язык сценариев JavaScript.

Последний стандарт HTML5 (по состоянию на июнь 2015 г. еще не принят окончательно) совместно с последним стандартом CSS3 позволяет еще больше упростить создание и компоновку документов HTML, а внешний вид Web страницы сделать еще более привлекательным для пользователя за счет подключения видео, аудио и различных динамических эффектов.

Для редактирования документов HTML разработано большое количество программного обеспечения, от небольших программ с подсветкой кода до серьезных программных комплексов с интегрированной средой разработки (IDE). Также выпущено огромное количество литературы по теме Web разработки, а в Сети можно легко найти готовые шаблоны Web страниц и примеры кода. Небольшое количество этой информации представлено далее. Существующий стандарт HTML, как язык разметки документа для размещения компонентов HTML-страницы относительно друг друга в рабочей области окна браузера не свободен от некоторых недостатков. Самыми главными из них являются следующие:
  1. В рамках языка HTML не удается достичь точного позиционирования таких компонентов страницы как текст, графика и различные встроенные приложения (видео, аудио и т.д.). Это связано с тем, что размер и границы каждого из этих компонентов в рамках HTML-разметки задаются с разной степенью точности. Размер графики и приложений можно задать с точностью до пиксела. Размеры текстовых блоков вычисляются браузером исходя из относительного размера шрифта по умолчанию.
  2. Форматирование элементов встраивается непосредственно в HTML теги в виде их атрибутов. Это приводит к значительному увеличению объема кода при использовании одинаковых стилей форматирования во многих местах страницы.
  3. В рамках HTML не всегда возможно полностью задать все детали внешнего вида некоторых компонентов страницы.
  4. Отсутствие средств, позволяющих легко менять внешнее оформление Web-документов без переделки самого документа.
Язык разметки CSS (от англ. Cascading Style Sheets - каскадные таблицы стилей) был разработан с целью устранить указанные недостатки HTML. Использование CSS дает возможность задать форматирование блока текста или изображения, а затем применять созданный стиль многократно в любом подходящем месте документа. Также стандарт CSS позволяет подключить дополнительные файлы, содержащие оформление желаемых компонентов Web-страницы, и таким образом, отделить компоновку и текстовое содержание документа от его внешнего представления. Логическая структура документа определяется элементами HTML-разметки, в то время как форма представления каждого из этих элементов задается CSS-дескриптором, соответствующим конкретному элементу. JavaScript – встраиваемый язык для программного доступа к объектам приложений. Наиболее широкое применение находит в браузерах как язык сценариев для придания интерактивности Web-страницам. Изначально JavaScript был разработан для реализации возможности производить какие-то действия, например, изменять оформление определенных элементов Web-страницы или показывать диалоговое окно с сообщением, в ответ на действия пользователя, обычно это работа с манипулятором типа "мышь" или нажатие определенной клавиши на клавиатуре.

Современный стандарт JavaScript для расширения своих возможностей активно использует библиотеки функций. Одна из них – JQuery. Эта библиотека с открытым исходным кодом, широко применяющаяся во всем мире, содержит в себе все основные функции по взаимодействию кода JavaScript и элементов HTML.

Хотя современные браузеры и дают возможность пошаговой отладки JavaScript-кода, но для серьезной работы желательно иметь специальную среду разработки, предоставляющую расширенные функции редактирования и отладки кода. Аббревиатура PHP расшифровывается как PHP: Hypertext Preprocessor – намек на обработку гипертекста. При этом PHP внутри формулировки – Personal Home Page, всего лишь "личная домашняя страница". Все это дает не слишком много информации для понимания назначения PHP. В действительности, PHP – это язык, предназначенный для быстрого создания динамических Web-страниц, т.е. таких, которые изменяются в зависимости от запросов браузера, с которым работает пользователь. Например, пользователь вводит свои данные на форме регистрации и нажимает кнопку с надписью "Отправить", "Зарегистрироваться" или что-то подобное. После проверки правильности заполнения формы на сервер отправляется запрос на получение данных о текущем пользователе или о его регистрации.

В официальной документации PHP определяется как server-side HTML-embedded scripting language, что означает "встраиваемый в HTML скриптовый язык с обработкой на сервере". Говоря проще, PHP это язык программирования, специально разработанный для написания Web-приложений (сценариев), исполняющихся на Web-сервере, в отличие, например, от сценариев, написанных на JavaScript, которые исполняются внутри браузера, т.е. на стороне клиента. Синтаксис PHP имеет общие черты с синтаксисом Java (возможность создания системы классов, ООП) и JavaScript (переменные со свободным объявлением типов), но имеет свои особенности. Для программирования на PHP тоже существуют свои среды разработки, а также обширная литература и соответствующая информация в Сети по этой теме. SQL (от англ. structured query language – "язык структурированных запросов") был создан как средство формирования запросов к реляционным базам данных (БД). Эти запросы обычно включают:
  • организацию данных в таблицах;
  • обновление данных, т.е. добавление в БД новых данных, удаление и изменение уже имеющихся;
  • чтение данных – от имени пользователя или прикладной программы
  • извлечение данных, удовлетворяющих группе критериев, из БД;
  • управление доступом на чтение и изменение данных;

    • Довольно часто на Web-серверах присутствует база данных, в которых может храниться самая разнообразная информация, такая как личные данные пользователей, научные статьи, каталоги изображений, товаров и др. Запросы к базе данных могут формироваться непосредственно на сервере с помощью языка сценариев PHP. Выполняются запросы посредством системы управления базой данных (СУБД). Одной из наиболее популярных СУБД для Web-серверов является MySQL. Как и PHP, она является продуктом свободного пользования. В настоящее время MySQL превратилась в полноценную технологию, входящую в состав многих современных наиболее посещаемых интернет-ресурсов.
